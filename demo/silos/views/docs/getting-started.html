Getting Started

This document describes the lifecycle of SiloJs.

Silo Modules

Silo js will first render content within a silo module.
The following demonstrates how to initiate a silo module:

Silo Module Configuration

<body silo="Demo"></body>

Yes, it is that simple.
Multiple silo elements may exist simultaniously, however, silo modules may not be nested. A "silo" is identified by the silo attribute and can be attached to any dom element.

Silo Controllers

Controllers allow Silo Markup Language to access properties and methods of any javascript object.
Controllers bridge the gap between your javascript code and html markup.
A Silo Controller has two main properties; markup and object.
The Silo Markup Language (SML) extends the silo namespace to html documents to expose all of it's features.

<body silo="Demo"> <silo:controller src="Navigation"> <h1>{{title}}</h1> </silo:controller> </body>

Silo Includes & Views

HTML content from external source is loaded by the Silo.View object.
Silo Markup Language extends the silo:include and silo:view directives for loading external markup;

<body> <silo:include src="http://somedomain.com/resource.html"></silo:include> </body>

Optionally, you can set the silo path as a base for view source.

<body silo="Demo" src="./demo"> <sio:view="getting-started.html"></sio:view> </body>

With the src attribute set on the silo object, SML will assume the path to be ./demo/views/geting-started.html

Contained Silo Markup

Silo markup that live outside controllers and within a Silo Component
Silo expression variables will test against Silo.scope

Global Silo Markup

Silo markup not contained within silo objects.
Silo expression variables will test against the window object.

<body> <script type="text/javascript"> var title = 'Silo Documentation'; var description = "Silojs bridges the gap between jsvascript and html"; </script> <h1>{{title}}</h1> <p>{{description}}</p> </body>
